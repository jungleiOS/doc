# 微信吃瓜热点文_案例素材_AI代码窃密事件

## 选题一句话

AI 编程工具不只是帮你写代码，也可能在你不知道的时候读取、打包、上传代码库、配置和密钥。

## 读者切身利益

程序员、创业者、产品团队、AI 工具用户的代码、API Key、商业机密和本地配置，都可能被工具静默读取或上传。

## 时间线

- 7月10日：苹果起诉 OpenAI，指控前员工和 OpenAI 硬件团队涉及商业机密问题。
- 7月11-12日：马斯克连续发帖攻击奥特曼，称其“善于诈骗”，暗示“该进监狱”。
- 7月10日：安全研究者逆向 Grok CLI，发现其静默上传用户工作目录快照。
- 7月13日凌晨：曝光后，Grok CLI 服务端远程切换开关，默认上传变为默认关闭，客户端二进制未更新。

## 三个核心瓜

### 瓜1：苹果起诉 OpenAI

- 人物/机构：苹果、OpenAI、Tang Tan、Chang Liu。
- 核心爆点：OpenAI 被指通过挖人和面试流程获取苹果硬件业务机密。
- 最离谱细节：面试时要求应聘者携带苹果实物零部件；前苹果工程师离职后仍访问服务器并下载超1000页机密文件。
- 可引用原话：OpenAI 的硬件业务从根上就烂了。
- 定性句：这不是普通挖人，是有组织、系统性的商业机密掠夺。

### 瓜2：马斯克骂奥特曼

- 表面行为：马斯克高强度攻击奥特曼和 OpenAI。
- 实际行为：自己旗下的 Grok CLI 被曝静默上传用户代码库。
- 讽刺点：一边骂别人偷，一边自己的工具也在偷用户代码。
- 定性句：全员恶人。

### 瓜3：Grok CLI 静默上传

- 个人/用户遭遇：用户只是要求 Grok 回复一个单词，Grok 仍上传两个代码库快照。
- 验证过程：逆向 Grok CLI，抓包确认每轮任务前后自动打包整个工作目录并上传到 xAI 的 Google Cloud 仓库。
- 离谱细节1：上传 tar.gz 快照、会话状态、配置和日志。
- 离谱细节2：跨项目扫描 Claude Code 配置，包括 `~/.claude.json`、AGENTS 规则和 30 多个 Skill 文件。
- 平台回应/暗改证据：曝光后服务端远程切开关，客户端没更新，二进制完全一样。
- 灵魂质问：如果合理、经过用户同意，为什么要偷偷关掉？

## 横向对比

- Claude Code、Codex、Gemini：保持本地运行，仅发送实际打开的文件。
- Claude Code 的负面点：系统提示词里有不可见 Unicode 分类标记。
- Grok CLI 的负面点：直接打包上传工作目录和配置。

## 可用比喻

- Claude Code 是人贱是贱，但至少没偷代码。
- Grok CLI 是复制你家钥匙，还顺走邻居家的钥匙。
- AI Agent 的权限等于管理员账户，但安全审计等于零。

## 可用金句

- 隐私裸奔，全看良心。
- AI Agent 能力越强，你的本地边界越脆弱。
- 希望有一天，我们不再需要靠逆向二进制文件，才能知道工具在背后做了什么。
- 那一天到来之前，保持警觉。

## 行动建议

1. 卸载 Grok CLI。
2. 检查 Claude Code 版本，特别关注被通报版本。
3. 轮换所有 API Key。
