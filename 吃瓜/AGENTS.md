# 吃瓜写作约束

适用范围：`吃瓜/` 目录下的模板、素材、核验报告、成文稿。

先遵守根目录 [AGENTS.md](/Users/jiangjun/doc/AGENTS.md) 的项目级禁令。
在此基础上，`吃瓜/` 目录额外遵守以下专项规则。

## 硬规则

1. **瓜一定要保熟。**
2. **先核验，后成文。** 没有核验，不得直接写正式吃瓜稿。
3. **必须有实锤证据。** 结论性表述必须能对应到截图、原话、时间线、官方回应、页面变更记录、公开链接等证据。
4. **无实锤，不下结论。** 不能把猜测、网传、二手转述写成既成事实。
5. **证据不闭环，降级表述。** 只能写“疑似”“有网友质疑”“目前能确认的是”，不能写“就是”“实锤”“已经证明”。

## 证据要求

每篇热点稿在动笔前，至少完成以下核验：

1. **时间线核验**：关键事件发生顺序明确，日期尽量具体到天，必要时具体到时分。
2. **来源核验**：优先一手来源，其次权威转述；尽量避免只依赖单条自媒体截图。
3. **原话核验**：争议性表述尽量找到原始上下文，避免断章取义。
4. **变更核验**：如果写“删帖、改口、暗改”，必须有前后对比证据。
5. **影响核验**：如果写“泄露、作恶、封号、骗钱”等重词，必须说明具体影响对象和影响方式。

## 文件要求

每个热点选题原则上配套三份文件：

1. `微信吃瓜热点文_案例素材_XXX.md`
2. `微信吃瓜热点文_案例素材_XXX_核验报告.md`
3. `微信吃瓜文_XXX.md`

规则：

1. 素材文件放事实、线索、截图说明、原话、链接。
2. 核验报告放证据链、已确认事实、未确认部分、风险点。
3. 成文稿只能使用核验报告里已经确认的内容。

## 写作红线

1. 不得把“网友爆料”直接写成事实。
2. 不得把情绪判断写得大于证据本身。
3. 不得为了凑“三个瓜”硬补未经核实的案例。
4. 不得使用“实锤、石锤、确定、已经证实”这类词，除非证据链足够完整。
5. 不得省略关键信息来源，只留下结论。

## 推荐流程

1. 先确认选题不触发根目录 `AGENTS.md` 的政治禁区。
2. 再建素材文件，收集线索。
3. 再做核验报告，区分“已确认 / 未确认 / 无法确认”。
4. 只有在证据足够时，才写正式稿。
5. 如果证据不足，就写成“热点整理 / 疑点梳理”，不要写成“定性揭露”。
